# История изменений

## [Неопубликовано] - 2024

### Добавлено

- **Обработка каждого N-го кадра**: Параметр `frame_skip` позволяет обрабатывать не все кадры, а только каждый N-й, что снижает нагрузку на процессор
- **Интерполяция при пропадании меток**: При временной потере меток система использует последнюю известную позу для интерполяции (до `max_lost_frames` кадров)
- **Улучшенный оффлайн визуализатор**:
  - Поддержка номеров кадров в траектории
  - Улучшенное отображение информации о точке (время, координаты, угол, номер кадра)
  - Более информативный интерфейс с выделенным блоком информации
- **Скрипт обработки последовательности изображений**: `process_image_sequence.py` для оффлайн обработки папки с изображениями
- **Документация архитектуры**: Файл `ARCHITECTURE.md` с подробными диаграммами работы системы (Mermaid и ASCII схемы)

### Изменено

- Траектория теперь сохраняет номер кадра в CSV/JSON формате
- Улучшена обработка ошибок при пропадании меток
- Обновлена конфигурация `tracker.yaml` с новыми параметрами

### Исправлено

- Исправлены XML теги в `package.xml` файлах
- Устранены дубликаты параметров в `tracker.yaml`
- Улучшена обработка граничных случаев при детекции меток

## [Начальная версия]

### Реализовано

- Gazebo симуляция с роботом и внешней камерой
- Tracker node для детекции цветных меток и вычисления позы робота
- Преобразование пиксельных координат в мировые через camera intrinsics + TF + ray-plane intersection
- Публикация позы робота и траектории в ROS топики
- Оффлайн визуализатор траектории с интерактивным ползунком
- Сохранение траектории в CSV/JSON формате
- RViz конфигурация для визуализации
- Документация проекта (README, INSTALL, QUICKSTART, PROJECT_STRUCTURE)
- Скрипт установки зависимостей
